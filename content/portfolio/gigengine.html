GigEngine

Introduction

Introducing GigEngine, a three-month school assignment undertaken by groups of three, aimed at crafting a functional engine and leveraging it to create a demo game. This project embraces a modern approach, incorporating essential features to provide valuable insights into engine architecture and design decisions, making it an invaluable resource for aspiring developers.

Key Features

  • Scene Graph with Quaternions for efficient spatial representation
  • In-Game UI for immersive user interaction in both ScreenSpace and WorldSpace environments
  • Skeletal Animation for lifelike character movements
  • Scripting capabilities through Lua for flexible and dynamic behavior within the game world
  • Serialization functionality for seamless data storage and retrieval
  • Integration of the Bullet physics engine for realistic interactions and simulations
  • Intuitive Editor powered by ImGui for streamlined content creation and manipulation
  • Advanced features like Shadow Mapping for enhanced visual fidelity

Project Details

GigEngine aims to provide valuable insights into engine architecture and design decisions. Explore its features and ongoing development progress on GitHub.

Although ongoing, GigEngine already encompasses various advanced features akin to professional-grade graphics engines. With an emphasis on clarity, performance, and comprehensibility, GigEngine's codebase aims to provide valuable insights into engine architecture and design decisions, making it an invaluable resource for aspiring developers.

While GigEngine continues to evolve, most of its feature set is readily available for exploration on its GitHub repository, inviting enthusiasts and learners alike to delve into the intricacies of graphics engine development.
